# 專案開發規範

此檔案定義開發此專案時必須遵守的規則，AI 助手在進行任何修改時應主動遵循。

---

## 響應式設計 (Responsive Design)

- **所有 UI 元件必須同時考慮手機 (寬度 < 640px) 和桌面裝置**
- Modal / 彈窗必須在手機上置中顯示，且不會被遮擋或超出螢幕
- 按鈕間距在手機上要有足夠的觸控範圍 (最小 44px)
- 使用 Tailwind 的響應式前綴 (`sm:`, `md:`, `lg:`) 來處理不同螢幕尺寸
- 完成 UI 修改後，需考慮在 375px (iPhone SE) 和 1280px (桌面) 寬度下的表現

---

## 使用者體驗 (UX)

- 刪除操作必須有確認提示，避免誤觸
- 提供「復原」機制給可能誤操作的重要動作
- 表單錯誤提示應清楚且靠近相關欄位

---

## 語言與文案

- 所有使用者可見的文字使用**繁體中文**
- 按鈕文字簡潔明瞭 (如「儲存」、「取消」、「確認刪除」)
- 錯誤訊息要友善且具體

---

## 程式碼風格

- 新功能應獨立成可重用的元件或 Hook
- 大型元件應拆分成小型子元件
- 使用 TypeScript 嚴格型別
- Git commit 訊息使用英文，格式：`feat:` / `fix:` / `refactor:`

---

## 資料與預設

- 內建資料 (如卡片範本、加碼預設) 應放在 `src/data/` 目錄
- 預設資料結構需有完整的 JSDoc 註解，方便未來維護
